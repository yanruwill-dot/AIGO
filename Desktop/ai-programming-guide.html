一、Claude Code 是什么？

简单说，它是一个命令行 AI 助手，能：

  • 读懂你的代码——自动理解项目结构、文件关系、代码逻辑
  • 帮你写代码——新增功能、修复 Bug、重构代码
  • 执行命令——运行测试、构建项目、执行 Git 操作
  • 搜索分析——在代码库中搜索、分析依赖关系

和普通 ChatGPT 的区别

普通 ChatGPT

  • 手动复制粘贴代码
  • 不知道项目上下文
  • 代码要自己放到文件里
  • 不能帮你跑命令

Claude Code

  • 直接读文件，自动理解
  • 知道整个项目结构
  • 直接写入文件，改好就生效
  • 帮你执行命令，一气呵成

二、10 分钟安装 Claude Code

Step 1：安装

# 使用 npm 安装（推荐） npm install -g @anthropic-ai/claude-code # 或者使用 Homebrew (macOS) brew install claude-code

Step 2：登录认证

# 启动 Claude Code claude # 首次运行会提示登录 # 按照提示完成 Anthropic 账号认证即可

Step 3：在项目目录启动

# 进入你的项目目录 cd your-project # 启动 Claude Code claude

💡 小贴士：Claude Code 会自动扫描当前目录，理解你的项目结构。所以一定要在项目根目录启动！

三、Claude Code 常用命令

四、AI Coding Template 是什么？

好，Claude Code 你已经会用了。但问题来了：

  • "AI 生成的代码质量怎么保证？"
  • "怎么让 AI 理解复杂的业务需求？"
  • "多人协作时，AI 怎么保持一致？"
  • "上下文丢失了怎么恢复？"

这就是 AI Coding Template 要解决的问题——它是一套经过实战验证的 AI 协作开发框架。

核心解决的问题

  • Phase Gate 机制：每个阶段设置质量关卡，必须通过才能进入下一步
  • 独立评审：引入另一个 AI（如 GPT-4）做交叉评审，避免自审偏差
  • 标准化文档：持久化关键信息到文档，随时可以恢复上下文
  • 契约验证：用 contract_resolver 查询真实 API，不让 AI 瞎编

五、8 阶段工作流：从想法到上线

AI Coding Template 把开发过程拆成 8 个阶段，每个阶段都有明确的产出：

0

Foundation（基础设施）

配置项目环境、安装依赖、设置 CI/CD

1

Kickoff（启动）

明确功能目标、收集背景信息、写 Context 文档

2

Spec（规格）⭐ 最重要

写清楚"要做什么"，这是 AI 理解需求的核心

3

Demo（原型）

快速生成可交互的原型，验证交互逻辑

4

Design（设计）

技术方案设计：API、数据库、组件拆分

5

Code（编码）

AI 执行编码，人类 Review

6

Test（测试）

自动化测试 + 手动验收

7

Deploy（部署）

上线发布、监控告警

🎯 核心原则：先 Spec 后 Code。没有清晰的 Spec，AI 写出来的代码大概率不是你想要的。

七、实战演示：用 AI 开发一个功能

让我们用一个真实例子走一遍流程——给用户列表页添加搜索功能。

Step 1：创建功能

/new-feature user-search

这会自动生成：

docs/user-search/ ├── context.md ├── spec.md ├── design.md └── test-plan.md

Step 2：写 Spec（最关键）

在 spec.md 里写清楚需求：

# 用户搜索功能 Spec ## 功能描述 在用户列表页顶部添加搜索框，支持按用户名和邮箱实时搜索。 ## 用户故事 - 作为运营人员，我希望能快速找到特定用户，以便查看和处理其信息。 ## 功能需求 1. 搜索框位于列表页顶部 2. 支持按用户名模糊匹配 3. 支持按邮箱精确匹配 4. 输入时实时过滤，无需点击搜索按钮 5. 搜索结果高亮匹配文字 ## 边界情况 - 空搜索词：显示全部用户 - 无匹配结果：显示"未找到相关用户" - 搜索词过短（<2字符）：不触发搜索 ## 验收标准 - [ ] 搜索响应时间 < 200ms - [ ] 支持中英文搜索 - [ ] 移动端适配正常

Step 3：让 AI 读取并开始编码

请阅读 docs/user-search/spec.md，然后帮我实现这个搜索功能。 先给我一个技术方案，确认后再写代码。

Step 4：Review & 迭代

AI 写完后，你来 Review：

帮我检查一下这个搜索功能： 1. 性能有没有问题？是否需要 debounce？ 2. 中文搜索会不会有问题？ 3. 有没有 XSS 风险？

Step 5：通过 Phase Gate

/phase-gate

系统会自动检查：必需产出物是否齐全、质量检查是否通过。

九、常见问题 FAQ

Q：AI 写的代码质量靠谱吗？

A：AI 写代码的质量取决于你给它的 Spec 质量。Spec 写得越清晰，AI 产出越好。这就是为什么我们强调"先 Spec 后 Code"。另外，Phase Gate 机制会在每个阶段做质量检查。

Q：上下文长度超限怎么办？

A：用 /compact 命令压缩上下文，或者用 /end-day 保存进度后重新开一个会话。标准化文档的好处就是——上下文随时可以从文档恢复。

Q：AI 生成的代码有错怎么办？

A：直接告诉它哪里错了，让它改。AI 很擅长根据反馈迭代。记住：你是 Driver，AI 是 Copilot。

Q：多人同时开发同一个功能怎么办？

A：用 PM Driver 编排，或者按领域拆分。文档要及时同步到 Git，保证大家看到的 Context 一致。
